END POINT : /register METHOD : POST
Required Parameters :
email, firstname, lastname, password

All above mentioned parameters are string type

*If needed, extra parameters can be added*
Activation link is sent to mail. Activates account with cadre of manager
END POINT : /login METHOD : POST
Required Parameters :
username(email), password

All above mentioned parameters are string type.There is a forgot password option too.

*email is primary key*
returns json web token, email, cadre of account holder
END POINT : /adduser METHOD : PUT
Required Parameters :
email

cadre of user should be manager. account should already be created.

*Bearer token needs to be passed*
Acceptance mail is sent to user mail. On acceptance cadre is changed to user.
END POINT : /createuser METHOD : POST
Required Parameters :
email, firstname, lastname, password

creator should be at manager cadre. user should have an account. users account is created automatically

*Bearer token needs to be passed*
Activation mail is sent to users mail. On Activation user will have the ability to just view his manager leads and his colleagues
END POINT : /removeuser METHOD : DELETE
Required Parameters :
email(user), password(manager)

One should be at manager cadre to delete user. user is removed from list of users under manager.

*Bearer token needs to be passed*
returns response Message about the status of user deletion.
END POINT : /createlead METHOD : POST
Required Parameters :
email, name, status

Extra Parameters can be added. Needs to be of manager cadre. Adds lead to list of manager's leads.

*Bearer token needs to be passed*
returns leads list of the manager
END POINT : /editlead METHOD : PATCH
Required Parameters :
email, status

Edits lead's status details of manager. Needs to be at manager cadre.

*Bearer token needs to be passed*
returns leads list of the manager
END POINT : /makecontact METHOD : PUT
Required Parameters :
email as query parameter

user that needs to be made contact needs to be a lead. user is removed from leads list and added to contact.

*Bearer token needs to be passed*
returns contacts list of the manager
END POINT : /startservice METHOD : POST
Required Parameters :
name(service), email(customer), status

Should be at manager cadre to start a service. Extra Parameters can be added.

*Bearer token needs to be passed*
returns services list of the manager
END POINT : /editservice METHOD : PATCH
Required Parameters :
name(service), email(customer)

Should be at manager cadre to edit a service. edits status of the service.

*Bearer token needs to be passed*
returns services list of the manager
END POINT : /getdetails METHOD : GET
Required Parameters :
No Parameters required

If users cadre is user, details of the manager is returned

*Bearer token needs to be passed*
returns list of services, leads, contacts, services, users of the manager or users manager.
